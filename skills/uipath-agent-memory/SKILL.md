# UiPath Agent Memory

Add, remove, inspect, and seed UiPath low-code agent memory spaces. This skill covers standalone low-code Agent Builder projects and inline low-code agents embedded in Flow projects.

## When to Use This Skill

- Add a memory space to a UiPath low-code agent.
- Configure dynamic few-shot retrieval for a low-code agent memory feature.
- Seed memory items into an agent project.
- List, inspect, or remove low-code agent memory features or memory seed items.
- Fix a low-code agent whose memory binding is missing, stale, or not visible after refresh.
- Work on an inline low-code agent where memory must propagate to the parent Flow project's `bindings_v2.json`.

Do not use this skill for coded-agent runtime memory/checkpoint libraries, generic RAG/vector indexes, Context Grounding indexes, Data Fabric records, or Action Center task operations.

## Critical Rules

1. **Use `uip agent memory` for memory features** - Do not hand-author `features/{FeatureName}/feature.json` for routine changes. The CLI owns the memory feature shape, item type validation, metadata JSON validation, and generated binding inputs.
2. **Attach an existing memory space** - `uip agent memory add` attaches a platform `MemorySpace`; it does not create one. Discover the memory space before attaching it. If none exists, stop and ask the user to create or provide one.
3. **Discover even when values are supplied** - Always attempt `uip solution resource list --source remote --kind MemorySpace` before attaching. Treat user-provided memory names and folders as search inputs and fallback values only when discovery is blocked by auth or connectivity.
4. **Use folder paths for memory add/remove** - Pass the literal Orchestrator folder path, such as `Shared` or `Shared/Sales`, as `--folder-path`. Do not pass folder keys to `uip agent memory add`.
5. **Refresh, validate, and refresh solution resources** - After any memory change, run `uip agent refresh`, `uip agent validate`, and then `uip solution resource refresh --output json` from the solution root when a solution is present.
6. **Never edit `bindings_v2.json` directly** - The `memorySpace` binding is generated by `uip agent refresh`. If it is missing or stale, fix the feature with `uip agent memory`, then refresh again.
7. **Seed only non-sensitive examples** - Do not put secrets, credentials, raw PII, or regulated personal data into seed memory items. Seed items become project configuration.
8. **Inline agents need binding propagation** - For inline low-code agents, the final refresh must include `--inline-in-flow --bindings-target "<FLOW_PROJECT_DIR>/bindings_v2.json"` so `uip solution resource refresh` can discover the generated `memorySpace` binding.
9. **Use `--output json` on parsed CLI commands** - Any command whose output you parse or use programmatically must include `--output json`.

## Workflow

### 1. Identify the Agent Target

Find the agent project directory:

```bash
find . -maxdepth 5 -name agent.json
```

Use the directory containing `agent.json` as `<AGENT_PROJECT_DIR>`.

Classify the target:

| Target | Signals | Refresh mode |
|---|---|---|
| Standalone low-code agent | `agent.json`, `project.uiproj`, `entry-points.json` | `uip agent refresh "<AGENT_PROJECT_DIR>" --output json` |
| Inline low-code agent | `agent.json` inside a UUID-like child directory under a Flow project; parent has `*.flow` | `uip agent refresh "<INLINE_AGENT_DIR>" --inline-in-flow --bindings-target "<FLOW_PROJECT_DIR>/bindings_v2.json" --output json` |

If several agent projects exist, pick the one the user named. If the target cannot be inferred safely, ask which agent to modify.

### 2. Verify the CLI Surface if Needed

If command availability or flag names are uncertain, inspect help before mutating files:

```bash
uip agent memory --help --output json
uip agent memory add --help --output json
uip agent memory item add --help --output json
```

### 3. Discover the Memory Space

Always try discovery first:

```bash
uip solution resource list \
  --source remote \
  --kind MemorySpace \
  --search "<MEMORY_SPACE_NAME>" \
  --output json
```

Use the discovered row's `Name` as `--memory-space` and `Folder` as `--folder-path`.

If discovery fails because authentication or connectivity is unavailable, continue only when the user already supplied both memory space name and folder path. Report that discovery was attempted and blocked.

If the memory space should be tracked in the solution resources, add the remote resource after discovery:

```bash
uip solution resource add \
  --source remote \
  --kind MemorySpace \
  --name "<MEMORY_SPACE_NAME>" \
  --folder-path "<FOLDER_PATH>" \
  --output json
```

This tracks an existing platform resource; it does not create a new memory space.

### 4. Attach the Memory Space

Choose a short PascalCase feature name that describes the memory's purpose, such as `SupportRecall`.

```bash
uip agent memory add <FEATURE_NAME> \
  --memory-space "<MEMORY_SPACE_NAME>" \
  --folder-path "<FOLDER_PATH>" \
  --threshold 0.25 \
  --result-count 5 \
  --search-mode hybrid \
  --field userQuestion=1 \
  --path "<AGENT_PROJECT_DIR>" \
  --output json
```

Common options:

| Option | Purpose |
|---|---|
| `--memory-space` | Existing tenant memory space name |
| `--folder-path` | Folder path containing the memory space |
| `--reference-key` | Optional solution resource key, when already known |
| `--description` | Human-readable feature description |
| `--threshold` | Retrieval score threshold; default `0` |
| `--result-count` | Number of memory results; default `3` |
| `--search-mode` | `hybrid` or `semantic`; default `hybrid` |
| `--field name=weight` | Input field weighting; repeat for multiple fields |
| `--disable-dynamic-few-shot` | Attach without runtime retrieval |
| `--path` | Agent project directory; default `.` |

Use field weights that match real agent input schema fields. Do not invent fields that are absent from `agent.json`.

### 5. Seed Memory Items Only When Requested

Seed examples are optional. Add them only when the user explicitly provides seed content or asks for default examples.

```bash
uip agent memory item add <FEATURE_NAME> <ITEM_KEY> <ITEM_VALUE> \
  --memory-type episodic \
  --feedback-id "<FEEDBACK_ID>" \
  --metadata '{"source":"seed"}' \
  --path "<AGENT_PROJECT_DIR>" \
  --output json
```

Memory item types:

| Value | Numeric | Use |
|---|---:|---|
| `episodic` | `0` | General recall examples |
| `escalation` | `1` | Escalation-related memory |

Rules:

- Episodic items require `--feedback-id`.
- `--metadata` must be a JSON object, not an array, string, or number.
- Adding an item with an existing key updates that item.
- Do not seed secrets, credentials, raw PII, or sensitive customer content.

### 6. Refresh and Validate

Standalone low-code agent:

```bash
uip agent memory list --path "<AGENT_PROJECT_DIR>" --output json
uip agent memory item list <FEATURE_NAME> --path "<AGENT_PROJECT_DIR>" --output json
uip agent refresh "<AGENT_PROJECT_DIR>" --output json
uip agent validate "<AGENT_PROJECT_DIR>" --output json
```

Inline low-code agent:

```bash
uip agent memory list --path "<INLINE_AGENT_DIR>" --output json
uip agent refresh "<INLINE_AGENT_DIR>" --inline-in-flow \
  --bindings-target "<FLOW_PROJECT_DIR>/bindings_v2.json" \
  --output json
uip agent validate "<INLINE_AGENT_DIR>" --inline-in-flow --output json
```

From the solution root, after refresh:

```bash
uip solution resource refresh --output json
```

After refresh, inspect `bindings_v2.json` only to verify that a `memorySpace` binding exists. Do not edit it. If refresh fails due to authentication, leave generated files intact and report the exact failed command.

## Remove Memory

Remove a memory feature by feature name or ID:

```bash
uip agent memory remove <FEATURE_NAME> \
  --path "<AGENT_PROJECT_DIR>" \
  --output json
```

Remove by memory space name only when also passing the folder path:

```bash
uip agent memory remove "<MEMORY_SPACE_NAME>" \
  --folder-path "<FOLDER_PATH>" \
  --path "<AGENT_PROJECT_DIR>" \
  --output json
```

Remove a seed item by key or ID:

```bash
uip agent memory item remove <FEATURE_NAME> <ITEM_KEY> \
  --path "<AGENT_PROJECT_DIR>" \
  --output json
```

Refresh, validate, and refresh solution resources after removal.

## Generated Shape for Review Only

The CLI writes:

```text
<AGENT_PROJECT_DIR>/features/SupportRecall/feature.json
```

Expected shape:

```json
{
  "$featureType": "memorySpace",
  "id": "<uuid>",
  "referenceKey": null,
  "folderPath": "Shared",
  "name": "SupportRecall",
  "memorySpaceName": "support-memory",
  "description": null,
  "isEnabled": true,
  "dynamicFewShotSettings": {
    "isEnabled": true,
    "threshold": 0.25,
    "resultCount": 5,
    "searchMode": "hybrid",
    "fieldSettings": [
      {
        "id": "<uuid>",
        "name": "userQuestion",
        "weight": 1
      }
    ]
  },
  "items": [
    {
      "id": "<uuid>",
      "key": "customer-tier",
      "value": "gold",
      "memoryType": 0,
      "feedbackId": "<feedback-id>",
      "description": null,
      "metadata": {
        "source": "seed"
      }
    }
  ]
}
```

Use this shape only for review and troubleshooting. Do not recreate it manually.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| No `memorySpace` binding after refresh | Refresh was not run after memory edit | Run `uip agent refresh "<AGENT_PROJECT_DIR>" --output json` |
| Inline agent memory exists but solution refresh misses it | Binding was not propagated to the parent Flow project | Re-run inline refresh with `--bindings-target "<FLOW_PROJECT_DIR>/bindings_v2.json"` |
| `required option '--memory-type <type>'` | `item add` always requires a type | Add `--memory-type episodic` or `--memory-type escalation` |
| `required option '--feedback-id <feedbackId>'` | Episodic items require a source feedback ID | Pass `--feedback-id "<FEEDBACK_ID>"` |
| `Invalid memory-type value` | Unsupported type | Use `episodic`, `escalation`, `0`, or `1` |
| `Invalid metadata JSON` | Metadata is malformed or not an object | Pass a valid JSON object, such as `'{"source":"seed"}'` |
| Multiple features match the same memory space name | Remove/list by memory space name is ambiguous | Use the feature name, feature ID, or pass `--folder-path` |
| User expects this command to create a new memory space | `uip agent memory add` only attaches existing spaces | Stop and ask for an existing memory space name and folder |
