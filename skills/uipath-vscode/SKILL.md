# UiPath VS Code Extension Assistant

Build, debug, and manage UiPath coded automation projects directly in Visual Studio Code.

## When to Use This Skill

- User wants to **install** the UiPath extension for VS Code
- User wants to **create a new coded workflow project** from VS Code
- User wants to **edit coded workflows** (.cs files) with IntelliSense and type checking
- User wants to **debug a workflow** using VS Code's built-in debugger
- User wants to **add or update NuGet packages** from the VS Code terminal
- User wants to **validate and run** workflows without opening UiPath Studio
- User wants to **configure** the extension settings (Studio path, NuGet feeds)

## Quick Start

### Step 1 — Install the UiPath VS Code extension

1. Open VS Code
2. Press `Ctrl+Shift+X` to open Extensions
3. Search for **"UiPath"**
4. Install **UiPath: Automation Development** (publisher: UiPath)
5. Reload VS Code when prompted

Alternatively, install from the terminal:
```bash
code --install-extension UiPath.UiPathDevelopment
```

### Step 2 — Configure Studio path

The extension needs to know where UiPath Studio is installed. Set it in VS Code settings:

1. Press `Ctrl+Shift+P` → type **"UiPath: Configure Studio Path"**
2. Enter the path to your Studio installation:
   - Default: `C:\Program Files\UiPath\Studio`
   - Custom install: your specific path

Or set it in `.vscode/settings.json`:
```json
{
    "uipath.studioDir": "C:\\Program Files\\UiPath\\Studio",
    "uipath.nugetFeeds": [
        "https://api.nuget.org/v3/index.json",
        "https://uipath.pkgs.visualstudio.com/Public.Feeds/_packaging/UiPath-Official/nuget/v3/index.json"
    ]
}
```

### Step 3 — Create or open a project

**Create new project from VS Code:**
```bash
# In VS Code terminal (Ctrl+`)
uip rpa create-project \
  --name "MyAutomation" \
  --location "C:/Projects" \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --format json
```

Then open the project folder: `File → Open Folder → C:/Projects/MyAutomation`

**Open existing project:** `File → Open Folder → <PROJECT_DIR>`

## Creating Projects

### New coded workflow project

```bash
# From VS Code integrated terminal (Ctrl+`)
uip rpa create-project \
  --name "InvoiceProcessor" \
  --location "C:/Projects" \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --format json

# Open in VS Code
code "C:/Projects/InvoiceProcessor"
```

### New test automation project

```bash
uip rpa create-project \
  --name "InvoiceProcessor.Tests" \
  --location "C:/Projects" \
  --template-id TestAutomationProjectTemplate \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --format json
```

### Project structure after creation

```
InvoiceProcessor/
├── .vscode/
│   ├── launch.json          ← Debug configurations (auto-generated by extension)
│   └── settings.json        ← UiPath extension settings
├── .codedworkflows/         ← Auto-generated (do not edit)
├── Main.cs                  ← Main workflow entry point
├── Main.cs.json             ← Metadata for Main.cs
├── project.json             ← Project configuration and dependencies
└── project.uiproj           ← Project descriptor
```

## Adding Workflow Files

### Create a new coded workflow file

1. **Via VS Code:** Right-click the project folder → **New File** → name it `ProcessInvoice.cs`
2. Use the coded workflow template:

```csharp
using UiPath.CodedWorkflows;

namespace InvoiceProcessor
{
    public class ProcessInvoice : CodedWorkflow
    {
        [Workflow]
        public void Execute(string invoicePath)
        {
            Log($"Processing: {invoicePath}");
            // Add automation logic here
        }
    }
}
```

3. Create the companion metadata file `ProcessInvoice.cs.json`:

```json
{
    "DisplayName": "ProcessInvoice",
    "Arguments": [
        {
            "Name": "invoicePath",
            "Direction": "In",
            "Type": "System.String, mscorlib, Version=4.0.0.0, Culture=neutral"
        }
    ]
}
```

4. Validate the file:

```bash
uip rpa validate \
  --file-path "ProcessInvoice.cs" \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --format json
```

## Managing Activity Packages

### Search for available packages

```bash
# Search NuGet for UiPath packages
uip rpa get-versions \
  --package-id UiPath.Excel.Activities \
  --include-prerelease \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --format json
```

### Install a package

```bash
# Install Excel activities
uip rpa install-or-update-packages \
  --packages '[{"id":"UiPath.Excel.Activities"}]' \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --format json

# Install multiple packages
uip rpa install-or-update-packages \
  --packages '[{"id":"UiPath.Excel.Activities"},{"id":"UiPath.Mail.Activities"},{"id":"UiPath.System.Activities"}]' \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --format json
```

### Inspect a package's available activities

```bash
uip rpa inspect-package \
  --package-id UiPath.Excel.Activities \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --format json
```

### VS Code IntelliSense for installed packages

After installing a package, the extension triggers Studio to generate local API docs at:
```
<PROJECT_DIR>/.local/docs/packages/<PackageId>/
```

These docs power IntelliSense suggestions for that package's services and methods.

## IntelliSense & Code Completion

The UiPath VS Code extension provides:

| Feature | How to access |
|---|---|
| Service method completion | Type `excel.` → see all Excel methods |
| Parameter hints | Inside method call parentheses — `Ctrl+Shift+Space` |
| Quick fix suggestions | Hover over red squiggle → **Quick Fix** |
| Go to definition | `F12` on any UiPath activity or service method |
| Find all references | `Shift+F12` |
| Rename symbol | `F2` |

### Getting the best IntelliSense

1. Ensure packages are installed (`.local/docs/` must exist)
2. Open the project folder (not a single file)
3. Allow the OmniSharp language server to fully load (status bar shows "OmniSharp is running")
4. If IntelliSense is missing for a package: reinstall it to regenerate `.local/docs/`

```bash
# Force regenerate local docs
uip rpa install-or-update-packages \
  --packages '[{"id":"UiPath.Excel.Activities"}]' \
  --project-dir "<PROJECT_DIR>" \
  --format json
```

## Debugging Workflows

### Step 1 — Set up launch configuration

The extension auto-generates `.vscode/launch.json`. If missing, create it:

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "UiPath: Run Workflow",
            "type": "uipath",
            "request": "launch",
            "projectDir": "${workspaceFolder}",
            "entryPoint": "Main",
            "studioDir": "C:\\Program Files\\UiPath\\Studio"
        },
        {
            "name": "UiPath: Debug Workflow",
            "type": "uipath",
            "request": "launch",
            "projectDir": "${workspaceFolder}",
            "entryPoint": "Main",
            "studioDir": "C:\\Program Files\\UiPath\\Studio",
            "stopOnEntry": true
        }
    ]
}
```

### Step 2 — Set breakpoints

Click in the gutter (left margin) next to any line in a `.cs` file to set a breakpoint. The red dot indicates the breakpoint is active.

### Step 3 — Start debugging

- Press `F5` to start debugging
- Use the Debug toolbar: **Continue** (F5), **Step Over** (F10), **Step Into** (F11), **Step Out** (Shift+F11)
- Inspect variables in the **Variables** panel (left sidebar during debug)
- Evaluate expressions in the **Debug Console** (`Ctrl+Shift+Y`)

### Step 4 — Watch UiPath-specific values

In the Watch panel, add expressions to monitor:
```
item.SpecificContent["InvoiceId"]
extractionResult.Fields["Total"]?.Value
response.StatusCode
```

## Running and Validating

### Validate a single file

```bash
uip rpa validate \
  --file-path "ProcessInvoice.cs" \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --format json
```

### Validate the entire project

```bash
uip rpa validate \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --format json
```

### Run a specific workflow

```bash
uip rpa run-file \
  --file-path "ProcessInvoice.cs" \
  --project-dir "C:/Projects/InvoiceProcessor" \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --input '{"invoicePath":"C:/temp/invoice.pdf"}' \
  --format json
```

### Run all tests

```bash
uip rpa run-tests \
  --project-dir "C:/Projects/InvoiceProcessor.Tests" \
  --studio-dir "C:/Program Files/UiPath/Studio" \
  --format json
```

## Workspace Tips for VS Code

### Recommended VS Code extensions alongside UiPath

| Extension | Purpose |
|---|---|
| **C# Dev Kit** (Microsoft) | Enhanced C# IntelliSense, test explorer |
| **GitLens** | Inline git blame and history |
| **REST Client** | Test API calls directly in VS Code |
| **Error Lens** | Inline error messages without hovering |

### Multi-project workspace

When working on multiple UiPath projects, use a VS Code workspace file:

```json
// uipath-projects.code-workspace
{
    "folders": [
        { "path": "C:/Projects/InvoiceProcessor" },
        { "path": "C:/Projects/InvoiceProcessor.Tests" },
        { "path": "C:/Projects/LeadEnrichment" }
    ],
    "settings": {
        "uipath.studioDir": "C:\\Program Files\\UiPath\\Studio"
    }
}
```

Open with: `code uipath-projects.code-workspace`

### Tasks for common operations

Add to `.vscode/tasks.json` for quick access via `Ctrl+Shift+P → Tasks: Run Task`:

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "Validate Project",
            "type": "shell",
            "command": "uip rpa validate --project-dir \"${workspaceFolder}\" --studio-dir \"C:/Program Files/UiPath/Studio\" --format json",
            "group": "build",
            "presentation": { "reveal": "always" }
        },
        {
            "label": "Run Main Workflow",
            "type": "shell",
            "command": "uip rpa run-file --file-path Main.cs --project-dir \"${workspaceFolder}\" --studio-dir \"C:/Program Files/UiPath/Studio\" --format json",
            "group": "test"
        },
        {
            "label": "Pack Project",
            "type": "shell",
            "command": "uip solution pack --project-dir \"${workspaceFolder}\" --output-dir \"${workspaceFolder}/dist\" --format json",
            "group": "build"
        }
    ]
}
```

## Troubleshooting

| Problem | Cause | Fix |
|---|---|---|
| IntelliSense not showing UiPath methods | OmniSharp not loaded / `.local/docs/` missing | Wait for OmniSharp; reinstall the package |
| `uip` command not found in terminal | CLI not installed or not on PATH | Run `npm install -g @uipath/cli`; restart VS Code |
| Validate returns "Studio not found" | Wrong `studioDir` in settings | Set correct path via `UiPath: Configure Studio Path` |
| Breakpoints not hit during debug | launch.json `studioDir` incorrect | Update launch.json `studioDir` to match actual Studio path |
| Extension not activating on project open | `project.json` missing or malformed | Ensure the workspace root contains a valid `project.json` |
| Build errors on `using` statements | Package not in `project.json` | Install the package with `uip rpa install-or-update-packages` |
