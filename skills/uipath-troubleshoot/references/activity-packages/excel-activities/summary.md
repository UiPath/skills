| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| Invoke VBA — Trust Access to VBA Project Denied | High | Excel "Trust access to the VBA project object model" setting disabled; activity cannot inject the macro module | [invoke-vba-trust-access.md](./playbooks/invoke-vba-trust-access.md) |
| Invoke VBA — Cannot Run Macro / Code File Unreadable | Medium | External `.txt`/`.vba` code file missing, malformed, wrongly encoded, or not wrapped in a `Sub`/`Function` block | [invoke-vba-code-file-path.md](./playbooks/invoke-vba-code-file-path.md) |
| Invoke VBA — Entry Method Name Mismatch | High | `EntryMethodName` does not resolve to a `Sub`/`Function` declared in the code file (typo, parentheses appended, nested macro) | [invoke-vba-entry-method-name.md](./playbooks/invoke-vba-entry-method-name.md) |
| Invoke VBA — Parameter Type or Shape Mismatch | Medium | `EntryMethodParameters` is not a properly-built `IEnumerable<Object>`, arity is wrong, or values were typed inline in the property window | [invoke-vba-parameter-formatting.md](./playbooks/invoke-vba-parameter-formatting.md) |
| Invoke VBA — COM Interop Failure (0x80010100) | Medium | Excel busy, blocked by a hidden modal dialog, Excel.exe hung, or multiple/wrong-bitness Office installs | [invoke-vba-com-interop-failure.md](./playbooks/invoke-vba-com-interop-failure.md) |
| Excel Not Installed / Interop Unavailable | High | Any Classic activity/scope, or a Modern card that selected COM, fails while acquiring `Excel.Application` on a host without desktop Excel (`REGDB_E_CLASSNOTREG` or `BusinessException: Error opening workbook. Make sure Excel is installed.`). Per-activity discriminator table separates Classic Lookup Range, other Classic activities, Classic scope, Modern card, and non-matching Workbook activities, then selects install-vs-OpenXML migration | [excel-not-installed.md](./playbooks/excel-not-installed.md) |
| Lookup Range — Value Not Found (Active Filters) | Medium | Active AutoFilters hide the target row; activity returns null/empty silently and a downstream fault is the first symptom | [lookup-range-active-filters.md](./playbooks/lookup-range-active-filters.md) |
| Lookup Range — Invalid Range or Value | Medium | `Range` set to `""` instead of blank, malformed A1 reference, unescaped wildcards, or a text-vs-number type mismatch in the search value | [lookup-range-invalid-range.md](./playbooks/lookup-range-invalid-range.md) |
| Lookup Range — Silent Miss Against a Formula Cell | Medium | Target value is the *computed* result of an Excel formula; Interop reads a stale/null cached value or fails to re-evaluate, so the lookup returns null even though the displayed text matches | [lookup-range-formula-cells.md](./playbooks/lookup-range-formula-cells.md) |
| Workbook Locked / File In Use By Another Process | Medium | Any workbook-opening activity - Read Range and the read/write family, Lookup Range (`ExcelLookUpRange` / `LookUpRangeX`), or the surrounding scope - cannot acquire the file because another process holds an exclusive lock (`System.IO.IOException: The process cannot access the file '<path>' because it is being used by another process.`, or `COMException 0x800A03EC` / `The file '<name>' is locked for editing by '<user>'` under COM). Causes: user editing in Excel UI, orphan `EXCEL.EXE` from a prior job, network-share lock on a different host, concurrent Robot jobs, AV/indexer/sync client, or a hidden Excel instance in another session. Per-activity discriminator table redirects sheet/file/lookup-result errors to sibling playbooks | [workbook-file-locked.md](./playbooks/workbook-file-locked.md) |
| Read Range — Sheet With Specified Name Does Not Exist | Medium | The workbook opened successfully but the configured `SheetName` does not match any sheet in the workbook (`UiPath.Excel.BusinessException: The sheet with the name '<name>' does not exist.`). Causes: typo, case mismatch on the OpenXML provider, sheet renamed or deleted upstream, leading/trailing whitespace, look-alike Unicode characters, or a dynamic `SheetName` expression resolving to the wrong value at runtime | [read-range-sheet-not-found.md](./playbooks/read-range-sheet-not-found.md) |
| Read Range — Workbook File Or Directory Not Found | Medium | The configured `WorkbookPath` does not resolve to an existing file at runtime (`System.IO.FileNotFoundException: Could not find file '<path>'.` or `System.IO.DirectoryNotFoundException`). Causes: file moved/deleted upstream, UNC share unreachable, relative path resolved against wrong CWD, drive letter not mapped under the Robot's session, OneDrive/SharePoint Files-On-Demand placeholder, or extension/casing mismatch on case-sensitive shares | [read-range-file-not-found.md](./playbooks/read-range-file-not-found.md) |
| NullReferenceException / TargetInvocationException From Inside An Excel Activity | Medium | An Excel activity faults with `System.NullReferenceException` (or `System.Reflection.TargetInvocationException` / provider wrappers) thrown from inside the activity. Surface A - sheet / range / scope resolution: missing sheet name, undefined named range, or activity outside any Excel scope (Lookup Range null-faults here instead of throwing a `BusinessException`); concrete verification path. Surface B - parsing internals (Read Range and other content-parsing activities): Microsoft Purview / AIP sensitivity label blocking the Robot user, workbook structural corruption (often from non-Excel writers like `openpyxl`), broken named range or formula reference (`#REF!`), unsupported OpenXML feature requiring COM, or heavy conditional formatting / parser scale limit - low-certainty, error wording does not distinguish branches; diagnosis usually requires manual workbook inspection | [null-reference-from-activity.md](./playbooks/null-reference-from-activity.md) |
| Write Cell Failures | Medium | `Write Cell` (Classic Workbook, Classic `Excel Application Scope`, or Modern `Use Excel File`) fails to write a value or formula. Symptoms surface as `System.IO.IOException` (file lock), `UiPath.Excel.BusinessException: ... wrong format, or Excel is busy` (formula syntax OR Excel COM state thrash — same wording, two causes), `BusinessException: ... sheet with the name '<x>' does not exist`, `COMException` about protected sheet / read-only workbook, or `BusinessException: ... cell reference '<x>' is invalid`. Causes: workbook locked OR Classic/Modern scope conflict (Classic Workbook activity racing a surrounding Modern scope on the same file), formula syntax (comma vs. semicolon separators, unescaped inner quotes, add-in-dependent functions), loop-induced Excel COM thrash when Write Cell runs cell-by-cell inside a tight loop, sheet name mismatch (same diagnostic as read-range-sheet-not-found), protected sheet / workbook / Read-only Mode scope, or invalid A1 cell reference / unknown named range. Includes anti-patterns section rejecting blanket Kill-Process-for-EXCEL, Retry-Scope-around-bad-formula, and Continue-On-Error suppression | [write-cell-failures.md](./playbooks/write-cell-failures.md) |
| Append Range Failures | Medium | `Append Range` (classic scoped) / `AppendRangeX` (modern scoped) / `Append Range Workbook` (standalone) fails. Signatures: `BusinessException` referencing missing `Excel Application Scope` / `Use Excel File`; `COMException 0x800A03EC Application-defined or object-defined error`; `BusinessException: The sheet with the name '<x>' does not exist`; `IOException: cannot access the file`; `NullReferenceException` on the activity; `BusinessException` mentioning "hidden rows" (package v2.8.5+). Causes: activity variant doesn't match the surrounding container (scoped activity outside a scope, OR Workbook activity colliding with a scope on the same path), sheet name mismatch or unsupported file extension for the active provider, workbook locked or read-only, uninitialized source DataTable, column schema mismatch between source DataTable and target sheet (count / order / types — appended by POSITION, not by header name), or hidden rows in the target append region. Includes anti-patterns section rejecting Delay-as-fix, bare-Try-Catch suppression, switch-to-Workbook-to-silence-scope-error, and pre-2.8.5 downgrade-to-silence-hidden-rows | [append-range-failures.md](./playbooks/append-range-failures.md) |
| Delete Range Failures | Medium | `DeleteRange` (classic) / `DeleteRangeX` (modern) fails. Signatures: `ArgumentException: The range is invalid` / `The string argument cannot be empty`; `COMException` referencing the shift operation or `Application-defined or object-defined error` (`0x800A03EC`); `IOException` / `COMException` with "cannot access the file"; or no exception with wrong rows removed. Causes: activity outside its required scope (`Excel Application Scope` / `Use Excel File`), invalid or empty `Range` syntax, `ShiftCells`/`ShiftOption` conflict with merged cells / Excel Tables, workbook locked or read-only, or AutoFilter hiding rows in the target block. Includes anti-patterns section rejecting Delay-as-fix, bare-Try-Catch suppression, and Delete-Range-when-Clear-Range-was-meant | [delete-range-failures.md](./playbooks/delete-range-failures.md) |
| Excel Application Scope (Classic) COM-Interop Failures | Medium | A SINGLE Classic `Excel Application Scope` (`ExcelApplicationScope`, COM-only) fails at the COM layer on a host that DOES have Excel. Signatures: `BusinessException: Error opening workbook. Make sure Excel is installed.` with inner `COMException 0x8002801D TYPE_E_LIBNOTREGISTERED` (or `0x80040154` despite a confirmed install); `BusinessException: Failed opening the Excel file. Possible reasons: file is corrupt, already used by another process or password protected.`; `InvalidCastException` on `System.__ComObject` with `0x80004002 E_NOINTERFACE`. Causes: COM registration / type-library corruption after an Office upgrade or repair (Excel present, interop broken); workbook held by an orphan `EXCEL.EXE` from a crashed prior job or a user-opened instance; a COM add-in (commonly `UiPath.Integration.ExcelAddin`) corrupting the interop QueryInterface. Fixes: Office Online Repair / UiPath Repair Tool / targeted `TypeLib` cleanup; clear orphan processes (`Kill Process EXCEL` on dedicated unattended hosts only) and fix what leaves orphans; disable the offending COM add-in. Includes anti-patterns rejecting Delay-as-fix, bare-Try-Catch suppression, full Office reinstall over repair, unconditional `Kill Process EXCEL` on shared hosts, and Visible-True-to-debug. Cross-references the missing-Excel failure-mode playbook and the card playbook's path, multi-scope race, child-outside-scope, and sensitivity branches | [excel-application-scope-failures.md](./playbooks/excel-application-scope-failures.md) |
| Use Excel File / Excel Application Card Failures | Medium | The Modern scope container `ExcelApplicationCard` (Use Excel File) or Classic `Excel Application Scope` fails to open the workbook or govern its lifecycle after missing desktop Excel has been routed to `excel-not-installed.md`. Signatures: `ArgumentException: Excel File path is empty or not set` / `Illegal characters in path`; `COMException RPC_E_SERVER_UNAVAILABLE` (`0x800706BA`) / `RPC_E_DISCONNECTED` (`0x80010108`) / `CO_E_SERVER_EXEC_FAILURE` (`0x80080005`); `BusinessException` on a child activity about missing scope; `COMException` / `BusinessException` referencing sensitivity label or information protection. Causes: empty / null / illegal `WorkbookPath` from variable scope drift or upstream producer returning empty; COM/RPC race across multiple scopes without an `Excel Process Scope` wrapper (or a child macro tore down EXCEL.EXE); Modern child activity dropped outside any card (cross-cut to per-activity playbooks); Microsoft Purview / AIP sensitivity-label rejection on pre-v2.23.4 packages or v2.23.4+ without the SensitivityLabel/Operation properties configured. Includes anti-patterns section rejecting Delay-as-fix, bare-Try-Catch suppression, Visible-True-to-debug (a COM-forcing footgun), and package-downgrade-to-silence-sensitivity-label | [excel-application-card-failures.md](./playbooks/excel-application-card-failures.md) |
| Execute Macro / Run Spreadsheet Macro Failures | Medium | VBA macro invoked via `ExecuteMacro` (classic) or `Run Spreadsheet Macro` (modern) fails — most commonly `System.Runtime.InteropServices.COMException` with HRESULTs `0x80020009 DISP_E_EXCEPTION`, `0x800AC472`, or `0x80010108 RPC_E_DISCONNECTED`; sometimes a job hang with no exception when Excel surfaces a modal dialog. Causes: macro name not in workbook, VBA error inside the macro, macro tears down Excel (Workbooks.Close / Application.Quit), modal dialog (MsgBox / InputBox / debug break) blocking COM, macros disabled by Trust Center policy on the Robot host, concurrent COM access (STA apartment violation), or missing add-in / ActiveX dependency. Includes anti-patterns section rejecting Delay-as-fix and bare-Try-Catch suppression | [execute-macro-failures.md](./playbooks/execute-macro-failures.md) |
| Write Range Failures | Medium | `WriteRange` (classic) / `WriteRangeX` (modern) fails. Signatures: `NullReferenceException` on the activity; `IOException`/`COMException` "cannot access the file"; `BusinessException` mentioning "empty source" / "Ignore empty source" / "header"; `BusinessException` mentioning "hidden rows" / "hidden columns" (package v2.8.5+); `OutOfMemoryException` or `COMException 0x800A03EC`. Causes: uninitialized `DataTable` variable (preceding step skipped or variable scope drift), workbook locked or read-only (orphan EXCEL.EXE / Mark as Final / Modern scope `ReadOnly=True`), empty source DataTable with `ExcludeHeaders=False`, hidden rows or columns inside the target write range, oversized batches or formula-prefix characters (`=`, `+`, `-`, `@`) misinterpreted as broken formulas. Includes anti-patterns section rejecting Delay-as-fix, bare-Try-Catch suppression, ExcludeHeaders-to-silence-empty-source, and pre-2.8.5 downgrade-to-silence-hidden-rows | [write-range-failures.md](./playbooks/write-range-failures.md) |
