# Coded Apps Playbooks

Covers failures in UiPath Coded Web Apps and Coded Action Apps (`@uipath/uipath-typescript` SDK, `uip codedapp`). Primary evidence: the error signature the user reports (OAuth error param, HTTP status, CORS message, deploy output), `uipath.json` (what the app requests), and `uip admin external-apps get <client-id> --output json` (what the OAuth client allows). See [overview.md](./overview.md) for the runtime/auth model and evidence sources.

| Issue | Confidence | Description | Playbook |
|-------|:---:|-------------|----------|
| redirect_uri_mismatch / Login Loop | High | Browser lands on UiPath with `error=redirect_uri_mismatch`, or bounces back to login without completing. The `redirectUri` the app sends is not registered on the External Application (trailing-slash, wrong port, or missing production URL after deploy). | [redirect-uri-mismatch.md](./playbooks/redirect-uri-mismatch.md) |
| invalid_scope in Auth URL | High | Callback URL contains `error=invalid_scope`. A scope listed in `uipath.json` is not enabled on the External Application, so the authorize request is rejected before login completes. | [invalid-scope.md](./playbooks/invalid-scope.md) |
| API Call Fails 401 After Login | High | App loads, first SDK call to `api.uipath.com` returns `HTTP 401 Unauthorized`. Token lacks the scope for that API, or was minted before a scope was added, or expired. NOT a 403 (folder/permission) — see discriminator in playbook. | [api-401-after-login.md](./playbooks/api-401-after-login.md) |
| API Call Fails 403 After Login | High | App loads, SDK call to `api.uipath.com` returns `HTTP 403 Forbidden`. Token is valid but the target folder (`UIPATH_FOLDER_KEY`) is wrong/unset, the resource is in another folder, or the user lacks the Orchestrator role. NOT a 401 (scope) — see discriminator in playbook. | [api-403-folder-permission.md](./playbooks/api-403-folder-permission.md) |
| sdk.isAuthenticated() false After Callback | High | Browser returns with `?code=` in the URL but `sdk.isAuthenticated()` stays `false` and the app loops back to login. App checks auth without calling `sdk.completeOAuth()`, or does custom callback parsing. NOT redirect_uri_mismatch (a code DID come back). | [isauthenticated-false-after-callback.md](./playbooks/isauthenticated-false-after-callback.md) |
| Action App: Form Data Not Loading | Medium | Coded action app renders but the form is blank; `codedActionAppService.getTask()` returns null or throws (often silently). Missing error handling, opened outside Action Center, missing `@uipath/coded-action-app`, or published as `Web` not `Action`. | [action-app-form-data-not-loading.md](./playbooks/action-app-form-data-not-loading.md) |
| CORS Error on API Call | High | Console shows `blocked by CORS policy` / `No 'Access-Control-Allow-Origin' header`, request going to `cloud.uipath.com`. `baseUrl` points at the portal domain instead of the API subdomain. | [cors-error-wrong-base-url.md](./playbooks/cors-error-wrong-base-url.md) |
| 404 After Deploy / App Not Found | High | Deployed app URL returns `404` / blank page, or JS/CSS assets 404 in the network tab. Vite `base` is not `'./'`, or the client-side router basename is hardcoded instead of `getAppBase()`. | [deploy-404-app-not-found.md](./playbooks/deploy-404-app-not-found.md) |
