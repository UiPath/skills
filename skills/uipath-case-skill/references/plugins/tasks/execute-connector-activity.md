# Execute Connector Activity Task

Executes an action on an external system via a UiPath Integration Service connector (e.g., create a Jira issue, send a Slack message).

## Prerequisites

- `uip login` required
- A healthy IS connection must exist for the connector

## Discovery

```bash
uip case registry search "<service name>" --output json
uip case registry get-connection --key <connectorKey> --output json
```

| Info | How to find it |
|---|---|
| Connector type ID | `uip case registry search "<service name>"` |
| Connection ID | `uip case registry get-connection --key <connectorKey>` |
| Required input fields | `uip case tasks describe --type connector-activity --id <typeId> --connection-id <id>` |

---

## Implementation

### Why CLI Enrichment Is Required

Same as the connector trigger: the task's `data.uipath.context[].metadata.body.activityPropertyConfiguration.configuration` is a serialized snapshot of the IS designer's internal state (`=jsonString:{...}`) — produced by the connector's introspection API, not user-authored JSON. It carries hundreds of fields per activity (instanceParameters, fieldsContainer with full output/input field metadata, cached lookups, etc.). See [connector-trigger.md → Why CLI Enrichment Is Required](../triggers/connector-trigger.md) for the full breakdown.

What **is** hand-writable around the task — and you should write directly:

| Part | Hand-writable? | Notes |
|---|---|---|
| Task `id`, `elementId`, `displayName`, `description`, `isRequired`, `shouldRunOnlyOnce` | ✓ | Standard task skeleton |
| `type: "execute-connector-activity"` | ✓ | Literal value |
| `entryConditions` | ✓ | Same as any task |
| Root `Connection` binding pair | ✓ | See [variables/bindings](../variables/bindings.md) — `resource: "Connection"`, `propertyAttribute: "ConnectionId"` + `"folderKey"` |
| `data.uipath.serviceType` | ✓ | Connector activities use `"Intsvc.Activity"` |
| `data.uipath.context[]` metadata + `inputs[].body` parameters + `outputs[]._jsonSchema` | ✗ | Generated by `tasks add-connector` from connector backend |

## Step 1 — Preview Schema

```bash
uip case tasks describe \
  --type connector-activity \
  --id <typeId> \
  --connection-id <connectionId> \
  --output json
```

Note all fields in `inputs` where `requiredFields` is non-empty — these must be provided.

## Step 2 — Add the Enriched Task

```bash
uip case tasks add-connector <file> <stage-id> \
  --type activity \
  --type-id <typeId> \
  --connection-id <connectionId> \
  --display-name "Create Jira Issue" \
  --lane 0 \
  --output json
```

With pre-populated inputs:
```bash
uip case tasks add-connector <file> <stage-id> \
  --type activity \
  --type-id <typeId> \
  --connection-id <connectionId> \
  --display-name "Create Jira Issue" \
  --input-values '{"body": {"fields.project.key": "PROJ", "fields.issuetype.id": "10004"}}' \
  --output json
```

Capture the `TaskId` from the output.

## What Gets Written

```json
{
  "id": "t<8chars>",
  "elementId": "<stageId>-t<8chars>",
  "displayName": "Create Jira Issue",
  "type": "execute-connector-activity",
  "data": {
    "serviceType": "Intsvc.ExecuteConnectorActivity",
    "context": [
      { "name": "connectorKey", "value": "uipath-atlassian-jira", "type": "string" },
      { "name": "connection",   "value": "=bindings.<connectionBindingId>", "type": "string" },
      { "name": "resourceKey",  "value": "<connectionId>", "type": "string" },
      { "name": "folderKey",    "value": "=bindings.<folderKeyBindingId>", "type": "string" },
      { "name": "operation",    "value": "Create", "type": "string" },
      { "name": "objectName",   "value": "Issue", "type": "string" },
      { "name": "method",       "value": "POST", "type": "string" },
      { "name": "path",         "value": "/issues", "type": "string" },
      { "name": "_label",       "value": "Create Issue", "type": "string" },
      { "name": "metadata",     "type": "json", "body": { ... } }
    ],
    "inputs": [ { "name": "body", "type": "json", "body": { ... } } ],
    "outputs": [
      { "name": "response", "type": "jsonSchema", "source": "=response", "_jsonSchema": { ... } },
      { "name": "Error",    "type": "json",       "source": "=error" }
    ],
    "bindings": []
  },
  "entryConditions": [
    { "id": "Condition_<6chars>", "displayName": "Stage entered", "rules": [[{ "rule": "current-stage-entered", "id": "Rule_<6chars>" }]] }
  ]
}
```

Root bindings appended automatically: connection + folderKey binding entries.

## Step 3 — Bind Remaining Inputs

```bash
uip case var bind <file> <stage-id> <task-id> \
  --name body.fields.project.key --value "=vars.projectKey" --output json
```
