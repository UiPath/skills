# Agent File Format Reference

Complete specification for all files in a low-code agent project.

---

## Project File Structure

```
<AgentName>/
├── agent.json                              # Main agent definition ← edit this
├── project.uiproj                          # Project metadata
├── entry-points.json                       # Entry point declarations
├── flow-layout.json                        # Studio canvas state
├── .agent-builder/                         # DO NOT EDIT — generated artifacts
│   ├── agent.json                          # Normalized execution model
│   ├── bindings.json                       # Tool → resource bindings
│   └── entry-points.json                   # Copy of root entry-points.json
├── .project/
│   └── JitCustomTypes.json                 # JIT compilation metadata
├── evals/
│   ├── eval-sets/
│   │   └── evaluation-set-default.json     # Default test suite
│   └── evaluators/
│       ├── evaluator-default.json          # Output quality evaluator
│       └── evaluator-default-trajectory.json  # Behavior evaluator
└── resources/                              # Local copies of tool definitions
    └── <ToolName>/resource.json
```

---

## `agent.json`

The primary agent definition. Schema version: `1.1.0`.

### Top-level structure

```jsonc
{
  "version": "1.1.0",               // always "1.1.0"
  "type": "lowCode",                // always "lowCode" for low-code agents
  "projectId": "<uuid>",            // must match SolutionStorage.json ProjectId

  "settings": { ... },              // LLM + runtime settings
  "metadata": { ... },              // Internal tracking fields

  "messages": [ ... ],              // System + user prompts
  "inputSchema": { ... },           // JSON Schema for agent inputs
  "outputSchema": { ... },          // JSON Schema for agent outputs
  "resources": [ ... ]              // Tools, contexts, escalations
}
```

### `settings` object

```jsonc
{
  "model": "gpt-4o-2024-11-20",     // LLM model identifier
  "maxTokens": 16384,               // Max output tokens
  "temperature": 0,                 // 0 = deterministic, 1 = creative
  "engine": "basic-v2",             // Agent engine: "basic-v2" (standard)
  "maxIterations": 25,              // Max reasoning loops before stopping
  "mode": "standard"                // Execution mode
}
```

Available models: `gpt-4o-2024-11-20`, `gpt-4o-mini`, `claude-3-5-sonnet`, `gemini-1.5-pro` (varies by tenant entitlements).

### `metadata` object

```jsonc
{
  "storageVersion": "50.0.0",              // DO NOT edit — managed by platform
  "isConversational": false,               // always false for low-code agents
  "targetRuntime": "pythonAgent",          // always "pythonAgent" for low-code
  "showProjectCreationExperience": false   // UI flag for first-time onboarding
}
```

### `messages` array

Defines system prompt and user prompt template.

```jsonc
[
  {
    "role": "system",
    "content": "You are a helpful assistant that answers questions accurately.",
    "contentTokens": []        // Auto-generated by Studio Web; leave as [] in source
  },
  {
    "role": "user",
    "content": "{{userInput}}", // References inputSchema property name in {{ }}
    "contentTokens": []
  }
]
```

The `role` values are `"system"` (first message, agent instructions) and `"user"` (second message, input template).

### `inputSchema` object

JSON Schema (draft-07) for the agent's input parameters.

```jsonc
{
  "type": "object",
  "title": "Agent Inputs",
  "properties": {
    "paramName": {
      "type": "string",           // string | number | boolean | object | array
      "title": "Display Name",
      "description": "What this parameter is for"
    }
  },
  "required": ["paramName"]       // optional: mark required params
}
```

**The property names in `inputSchema.properties` must match the `{{placeholder}}` names in the user message content.**

### `outputSchema` object

```jsonc
{
  "type": "object",
  "properties": {
    "content": {
      "type": "string",
      "description": "The agent's primary response"
    },
    "additionalField": {
      "type": "number",
      "description": "An additional output value"
    }
  }
}
```

### `resources` array

Polymorphic array; each item has a `$resourceType` discriminator.

#### Tool resource (`$resourceType: "tool"`)

```jsonc
{
  "$resourceType": "tool",
  "id": "<uuid>",                  // Stable tool ID; generate once
  "referenceKey": "<uuid>",        // Reference key for binding lookup
  "name": "ToolDisplayName",
  "type": "process",               // See type table below
  "location": "external",          // "external" | "solution"
  "description": "What this tool does (shown to LLM for tool selection)",
  "inputSchema": {
    "type": "object",
    "properties": {
      "param1": { "type": "string", "description": "First parameter" }
    },
    "required": ["param1"]
  },
  "outputSchema": {
    "type": "object",
    "properties": {
      "result": { "type": "string" }
    }
  },
  "isEnabled": true,
  "properties": {
    "processName": "MyProcess",
    "folderPath": "solution_folder"   // or actual path like "Shared"
  }
}
```

**`type` values for tools:**

| Value | Use when |
|-------|----------|
| `process` | Calling an RPA process (XAML workflow) in Orchestrator |
| `agent` | Calling another agent (solution-internal or external) |
| `integration` | Calling an Integration Service connector activity |
| `api` | Direct REST API call |
| `ixp` | Internal UiPath platform service |
| `mcp` | MCP (Model Context Protocol) server tool |

**`location` values:**

| Value | Meaning |
|-------|---------|
| `external` | Resource lives in Orchestrator, outside this solution |
| `solution` | Resource is another project within this same solution |

**`folderPath` values:**

| Value | When to use |
|-------|------------|
| `"solution_folder"` | Resource is part of this solution (deployed together) |
| `"Shared"` | Resource is in the Shared Orchestrator folder |
| `"Shared/SubFolder"` | Resource is in a specific Orchestrator subfolder |

#### Context resource (`$resourceType: "context"`)

```jsonc
{
  "$resourceType": "context",
  "contextType": "index",          // "index" | "attachments" | "dataFabricEntitySet"
  "indexName": "MyIndex",
  "folderPath": "solution_folder",
  "settings": {
    "query": {
      "variant": "dynamic"         // "dynamic" = query at runtime from user input
    },
    "retrievalMode": "semantic",   // "semantic" | "structured" | "deepRAG" | "batchTransform"
    "resultCount": 3,              // Number of chunks to retrieve
    "threshold": 0,                // Minimum relevance score (0–1)
    "fileExtension": "All"         // "All" | "pdf" | "docx" | etc.
  }
}
```

#### Escalation resource (`$resourceType: "escalation"`)

```jsonc
{
  "$resourceType": "escalation",
  "id": "<uuid>",
  "name": "Human Review",
  "description": "Escalate to a human reviewer when uncertain",
  "isEnabled": true,
  "channels": [
    {
      "name": "ActionCenter",
      "type": "ActionCenter",       // "ActionCenter" | "Email" | "Teams" | etc.
      "inputSchema": { ... },
      "properties": { ... }
    }
  ]
}
```

## `project.uiproj`

Minimal project metadata. For low-code agents this is always:

```json
{
  "ProjectType": "Agent",
  "Name": "MyAgentName",
  "Description": null,
  "MainFile": null
}
```

**`ProjectType` must be `"Agent"`** — this is what Studio Web and the packager use to identify low-code agent projects. Other valid types: `"Process"` (RPA XAML), `"Flow"`, `"BusinessAgent"`.

---

## `entry-points.json`

Declares the callable entry points with their I/O contracts.

```jsonc
{
  "$schema": "https://cloud.uipath.com/draft/2024-12/entry-point",
  "$id": "entry-points.json",
  "entryPoints": [
    {
      "filePath": "/content/agent.json",    // Path inside the package
      "type": "agent",                       // always "agent" for low-code agents
      "uniqueId": "<uuid>",                  // Stable ID; matches model.agentProjectId in flows
      "input": {
        "type": "object",
        "properties": {
          "userInput": { "type": "string" }
        }
      },
      "output": {
        "type": "object",
        "properties": {
          "content": { "type": "string" }
        }
      }
    }
  ]
}
```

The `input` and `output` schemas should mirror the agent's `inputSchema` and `outputSchema`.

## `.agent-builder/bindings.json`

Maps tool references to their Orchestrator resources. **Generated — do not edit.**

```jsonc
{
  "version": "2.0",
  "resources": [
    // Process tool binding
    {
      "resource": "process",
      "key": "MyProcess",           // Tool name from agent.json
      "value": {
        "name": {
          "defaultValue": "MyProcess",
          "isExpression": false,
          "displayName": "Process name"
        },
        "folderPath": {
          "defaultValue": "solution_folder",
          "isExpression": false,
          "displayName": "Folder path"
        }
      },
      "metadata": {
        "subType": "process",       // "process" | "Agent"
        "bindingsVersion": "2.2",
        "solutionsSupport": "true"
      }
    },

    // Connection (IS integration) binding
    {
      "resource": "connection",
      "key": "my-connector-connection-id",
      "value": {
        "connectionId": {
          "defaultValue": "<connection-uuid>",
          "isExpression": false,
          "displayName": "Connection ID"
        }
      },
      "metadata": {
        "bindingsVersion": "2.2"
      }
    },

    // Index (RAG) binding
    {
      "resource": "index",
      "key": "MyIndex",
      "value": {
        "name": {
          "defaultValue": "MyIndex",
          "isExpression": false,
          "displayName": "Index name"
        },
        "folderPath": {
          "defaultValue": "solution_folder",
          "isExpression": false,
          "displayName": "Folder path"
        }
      },
      "metadata": {
        "bindingsVersion": "2.2"
      }
    }
  ]
}
```

**`resource` field values:**

| Value | Resource type |
|-------|--------------|
| `process` | Orchestrator process (RPA or agent) |
| `connection` | Integration Service connection |
| `index` | Semantic search index |
| `memorySpace` | Agent memory space |
| `mcpServer` | MCP server |
| `app` | UiPath App (HITL) |
| `asset` | Orchestrator asset |
| `bucket` | Orchestrator storage bucket |
| `dataFabricEntity` | Data Fabric entity set |

## `evals/evaluators/evaluator-default.json`

Output quality evaluator (semantic similarity).

```jsonc
{
  "fileName": "evaluator-default.json",
  "id": "<uuid>",
  "name": "Default Evaluator",
  "description": "Evaluates semantic similarity between expected and actual output",
  "type": 5,                      // 5 = semantic similarity
  "category": 1,                  // 1 = output evaluation
  "prompt": "Given the expected output: '{{ExpectedOutput}}' and the actual output: '{{ActualOutput}}', score the similarity from 0 to 100.",
  "model": "same-as-agent",       // uses the agent's configured model
  "targetOutputKey": "*",         // "*" = evaluate all output fields
  "targetSubOutputKey": "",
  "createdAt": "2026-04-01T12:00:00Z",
  "updatedAt": "2026-04-01T12:00:00Z"
}
```

## `evals/evaluators/evaluator-default-trajectory.json`

Behavior/trajectory evaluator.

```jsonc
{
  "fileName": "evaluator-default-trajectory.json",
  "id": "<uuid>",
  "name": "Default Trajectory Evaluator",
  "type": 7,                      // 7 = trajectory evaluator
  "category": 3,                  // 3 = behavior evaluation
  "prompt": "Given the following context:\n- User Input: {{UserOrSyntheticInput}}\n- Expected Behavior: {{ExpectedAgentBehavior}}\n- Agent Run History: {{AgentRunHistory}}\nDid the agent behave as expected? Score 0-100.",
  "model": "same-as-agent",
  "targetOutputKey": "*",
  "targetSubOutputKey": ""
}
```

## UUID Generation

All UUIDs in agent.json must be stable v4 UUIDs. Generate with:
```bash
uuidgen          # Unix/macOS
[System.Guid]::NewGuid()  # PowerShell
node -e "console.log(require('crypto').randomUUID())"  # Node.js
```

Never reuse a UUID from one resource in another — every binding key must be unique within the project.
