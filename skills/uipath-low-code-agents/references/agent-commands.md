# Agent CLI Commands Reference

Commands for managing low-code agent projects split across two tools:
- **`uip low-code-agent`** — agent project scaffolding and artifact generation
- **`uip solution`** — all solution lifecycle: pack, bundle, upload, publish, deploy, activate

---

## Authentication

```bash
# Check login status
uip login status --output json

# Login interactively
uip login

# Login with client credentials (CI/CD)
uip login --client-id <ID> --client-secret <SECRET> --tenant <TENANT>

# Switch tenant
uip login tenant list --output json
uip login tenant set "<TENANT_NAME>"
```

## `uip low-code-agent init` — Scaffold agent project

Creates a low-code agent project at the given path. Creates **only** the agent project files — no solution files (`SolutionStorage.json`, `.uipx`, `resources/`). Those are created separately via `uip solution new`.

```bash
uip low-code-agent init <path> [options]
```

| Argument/Flag | Default | Description |
|--------------|---------|-------------|
| `<path>` | required | Target directory (relative or absolute). Agent name is derived from `basename(<path>)`. |
| `--model <model>` | `gpt-4o-2024-11-20` | LLM model |
| `--system-prompt <prompt>` | (generic) | Initial system prompt |
| `--force` | false | Overwrite non-empty directory |

**Files created at `<path>/`:**
```
agent.json                              # Main agent definition — edit this
project.uiproj                          # Project metadata (type: Agent)
entry-points.json                       # Entry point I/O declarations
flow-layout.json                        # Studio canvas state
.agent-builder/                         # Build artifacts — do NOT edit manually
│   agent.json                          # Execution model (regenerated by `build`)
│   bindings.json                       # Tool bindings (regenerated by `build`)
│   entry-points.json                   # Copy of root entry-points.json
.project/JitCustomTypes.json
evals/evaluators/evaluator-default.json
evals/evaluators/evaluator-default-trajectory.json
evals/eval-sets/evaluation-set-default.json
features/                               # (empty)
resources/                              # Local resource definition copies
```

**Output:**
```json
{
  "Result": "Success",
  "Code": "LowCodeAgentInit",
  "Data": {
    "Status": "Agent project created",
    "Path": "/abs/path/to/Agent",
    "Name": "Agent",
    "Model": "gpt-4o-2024-11-20",
    "ProjectId": "<uuid>",
    "NextSteps": "..."
  }
}
```

## `uip low-code-agent build` — Regenerate .agent-builder artifacts

Reads `agent.json` and regenerates all `.agent-builder/` files. **Run this every time you modify `agent.json`** (add/remove tools, change resources) before packing the solution.

```bash
uip low-code-agent build [path] [options]
```

| Argument/Flag | Default | Description |
|--------------|---------|-------------|
| `[path]` | `.` (current directory) | Agent project directory |
| `--dry-run` | false | Print what would be written without writing |

**What it generates:**

| File | From |
|------|------|
| `.agent-builder/agent.json` | Spreads `agent.json` + adds `id`, `name`, propagates `resources`/`features` |
| `.agent-builder/bindings.json` | Derived from `agent.json` resources (see mapping table below) |
| `.agent-builder/entry-points.json` | Copied from root `entry-points.json` |

**Binding derivation rules:**

| Resource in `agent.json` | `resource` field | `key` | `value` fields |
|--------------------------|-----------------|-------|----------------|
| `tool` / `type: "process"` | `"process"` | `tool.name` | `name`, `folderPath` from `properties` |
| `tool` / `type: "agent"` | `"process"` | `tool.name` | `name`, `folderPath`; `subType: "Agent"` |
| `tool` / `type: "integration"` | `"connection"` | `tool.referenceKey` | `connectionId` from `properties` |
| `context` / `contextType: "index"` | `"index"` | `ctx.indexName` | `name`, `folderPath` |
| `mcp` | `"mcpServer"` | `mcp.name` | `{}` |
| `escalation` | _(no binding entry)_ | — | — |

**Output:**
```json
{
  "Result": "Success",
  "Code": "LowCodeAgentBuild",
  "Data": {
    "Status": "Built successfully",
    "ProjectDir": "/abs/path/to/Agent",
    "AgentName": "Agent",
    "ResourceCount": 3,
    "BindingCount": 3,
    "Written": [".agent-builder/agent.json", ".agent-builder/bindings.json", ".agent-builder/entry-points.json"]
  }
}
```

## `uip solution` — All solution lifecycle commands

### Create solution

```bash
uip solution new "<SolutionName>" --output json
# Creates: <SolutionName>.uipx  +  SolutionStorage.json
```

### Add / remove agent project

```bash
uip solution project add <agentProjectDir> [<solutionFile>.uipx] --output json
uip solution project remove <agentProjectDir> [<solutionFile>.uipx] --output json
```

### Pack for Automation.Solutions (deployment)

```bash
uip solution pack <solutionDir> <outputDir> --output json
uip solution pack <solutionDir> <outputDir> -n "MySolution" -v "1.2.0" --output json
# → MySolution.1.2.0.zip in <outputDir>
```

### Bundle for Studio Web (upload)

```bash
uip solution bundle <solutionDir> --output json
uip solution bundle <solutionDir> -d <outputDir> -n "MySolution" --output json
# → MySolution.uis in <outputDir>
```

### Upload to Studio Web

```bash
uip solution upload <solutionDir>   # auto-bundles to .uis, then imports
uip solution upload <file>.uis      # imports directly
uip solution upload <file>.uipx     # imports from .uipx
# Auto-detects import vs overwrite based on whether solution already exists
```

### Publish package to Orchestrator

```bash
uip solution publish <outputDir>/MySolution.1.0.0.zip --output json
# → { PackageVersionKey, PackageName, PackageVersion }
```

### Deploy (install + auto-activate)

```bash
uip solution deploy run \
  --name "MySolution-Production" \
  --package-name "MySolution" \
  --package-version "1.0.0" \
  --folder-name "MySolution" \
  --folder-path "Shared" \
  --output json
# Polls until DeploymentSucceeded
# → { Status, DeploymentKey, PipelineDeploymentId, InstanceId }
```

| Flag | Required | Description |
|------|----------|-------------|
| `-n, --name <name>` | yes | Deployment name |
| `--package-name <name>` | yes | Solution package name |
| `--package-version <version>` | yes | Package version |
| `--folder-name <name>` | yes | Target Orchestrator folder name to create |
| `--folder-path <path>` | no | Parent folder path (e.g., `Shared`) |
| `--folder-key <key>` | no | Parent folder GUID (alternative to path) |
| `--config-file <path>` | no | Custom config JSON (from `uip solution deploy config`) |
| `--timeout <seconds>` | no | Polling timeout (default: 360) |

Terminal states: `DeploymentSucceeded`, `DeploymentFailed`, `ValidationFailed`

### Activate existing deployment

```bash
uip solution deploy activate "<deployment-name>" --output json
# → { Status, DeploymentName, InstanceId }
```

Terminal states: `SuccessfulActivate`, `FailedActivate`

### Check deployment status

```bash
uip solution deploy status <pipeline-deployment-id> --output json
# → { Status, DeploymentKey, ConfigurationKey, InstanceId, DeploymentResult }
```

### List deployments

```bash
uip solution deploy list --output json
uip solution deploy list --folder-path "Shared" --take 20 --output json
```

### Get deployment config (for custom overrides)

```bash
uip solution deploy config "MySolution" --output json
uip solution deploy config "MySolution" -d config.json  # save to file
# Edit config.json, then pass to deploy run:
uip solution deploy run ... --config-file config.json
```

### Uninstall deployment

```bash
uip solution deploy uninstall "<deployment-name>" --output json
# → { Status, DeploymentName, InstanceId }
```

Terminal states: `SuccessfulUninstall`, `FailedUninstall`

### List published packages

```bash
uip solution packages list --output json
```

## Complete lifecycle — end to end

```bash
# 1. Login
uip login status --output json

# 2. Create solution
uip solution new "MySolution" --output json

# 3. Scaffold agent project(s) inside solution directory
uip low-code-agent init ./MySolution/Agent --model gpt-4o-2024-11-20 --output json
uip low-code-agent init ./MySolution/Agent2 --model gpt-4o-2024-11-20 --output json

# 4. Link projects to solution
uip solution project add ./MySolution/Agent  ./MySolution/MySolution.uipx --output json
uip solution project add ./MySolution/Agent2 ./MySolution/MySolution.uipx --output json

# 5. Edit agent.json files: add tools, contexts, configure prompts
#    (see agent-file-format.md)

# 6. Rebuild .agent-builder/ after each edit
uip low-code-agent build ./MySolution/Agent  --output json
uip low-code-agent build ./MySolution/Agent2 --output json

# 7. Upload to Studio Web for visual development
uip solution upload ./MySolution --output json

# 8. (After visual work in Studio Web) pull back
#    Use Studio Web export or the solution upload flow in reverse

# 9. Pack for deployment
uip solution pack ./MySolution ./output -v "1.0.0" --output json

# 10. Publish to Orchestrator
uip solution publish ./output/MySolution.1.0.0.zip --output json

# 11. Deploy
uip solution deploy run \
  --name "MySolution-v1" \
  --package-name "MySolution" \
  --package-version "1.0.0" \
  --folder-name "MySolution" \
  --folder-path "Shared" \
  --output json
```

## Output format

All commands support `--output json`:

```json
{ "Result": "Success", "Code": "...", "Data": { ... } }
{ "Result": "Failure", "Message": "...", "Instructions": "..." }
```

Always use `--output json` when parsing command output in scripts or skills.
